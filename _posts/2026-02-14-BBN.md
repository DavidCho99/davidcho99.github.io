## Data Recategorization / Feature Engineering

### Purpose

### Why This Step Is Necessary
The original dataset contains many rare categories, particularly for cancer cell subtypes and detailed treatment combinations.  
Training a Bayesian network on such sparse categories can lead to:

- Unstable Conditional Probability Table (CPT) estimates due to very small sample counts  
- Poor generalization to unseen data  
- Overly complex network structures that are difficult to interpret  

By recategorizing variables into broader clinical groupings, we improve statistical stability, reduce overfitting, and enhance the interpretability of the learned Bayesian network.

## 4. Structure Learning

### Purpose

To learn the Directed Acyclic Graph (DAG) representing conditional dependencies among variables.

### Method
* Hill Climbing Search
* Bayesian Information Criterion (BIC) score

### Why This Step Is Necessary
* Hill Climbing efficiently explores large graph spaces
* BIC penalizes unnecessary complexity, reducing overfitting
* The learned structure provides probabilistic relationships suitable for clinical analysis



## 5. Bayesian Belief Network Construction

### Purpose

To construct a Bayesian Network using the learned structure.

### Why This Step Is Necessary

Some variables may be conditionally independent and thus have no edges.
Including isolated nodes ensures the network represents the full joint probability distribution.



```python
# 4. Construct Bayesian Belief Network (BBN)
# Initialize the Bayesian Network with the learned structure.
bbn_model = DiscreteBayesianNetwork(consistent_edges)

# Add isolated nodes (nodes with no edges) to the model if any
bbn_model.add_nodes_from(train_data.columns)
```

## 6. Parameter Learning

### Implementation Details

A helper function is defined to convert pgmpy’s `DiscreteFactor` output into a pandas DataFrame that resembles common Bayesian network output formats (e.g., bnlearn-style probability tables used in CS 3368).

```python
def query_to_df(q):
    """
    Convert pgmpy query result (DiscreteFactor) into a pandas DataFrame.
    Output columns: target variable states and their corresponding probabilities.
    """
    var = q.variables[0]
    states = q.state_names[var]
    probs = q.values

    out = pd.DataFrame({var: states, "prob": probs})
    return out
```

## Improving Structure Learning (Hill Climb Search )

### Purpose  
To learn the **Bayesian Network structure** (a Directed Acyclic Graph, DAG) that best explains dependencies in the lung-cancer dataset.

### Why This Step Is Necessary
Right now, the current Hill Climb implementation is essentially:
- “run hill climb with BIC and a max iteration cap”

That works, but it’s also the **most basic** version. In real clinical-style datasets (many variables + noisy categories), basic hill climb often:
- gets stuck in local optima
- learns unstable edges (small data changes → different graphs)
- creates overly dense graphs or spurious links
- becomes slow as the number of variables increases

So the next step to improve the project is to upgrade Hill Climb Search with better constraints, better scoring, and better robustness.

---

## What to Improve



### 1) Use max_indegree (limit parents per node)
#### Purpose  
Keep the learned graph simple + interpretable.

#### Why needed  
Without this, hill climb may create nodes with too many parents → CPT explosion:
- CPT size grows exponentially with number of parents  
- leads to overfitting + slow inference

Typical clinical BN practice: `max_indegree = 2~4`

---

### 2) Try alternative scoring methods
#### Options
- **BIC**: stronger penalty → simpler graphs (good default)
- **K2Score**: assumes complete data & discrete states; can behave differently
- **BDeu**: Bayesian score; can be smoother when counts are small

#### Why needed  
With sparse categories (even after recategorization), BIC can still be harsh/unbalanced.
Testing 2–3 scores is a standard improvement step.

---

### 3) Run multiple random restarts
#### Purpose  
Reduce chance of being stuck in a local optimum.

#### Why needed  
Hill climb is greedy. Starting from an empty graph once can miss better solutions.

Approach:
- run hill climb N times with different starting graphs
- keep the best-scoring graph
